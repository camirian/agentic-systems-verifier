## ğŸŒ¿ Branching Strategy

We follow a standard Feature Branch workflow:

1.  **Fork** the repository.
2.  Create a **Feature Branch** from `main`:
    ```bash
    git checkout -b feature/my-new-feature
    ```
3.  **Commit** your changes:
    ```bash
    git commit -m "feat: Add amazing new feature"
    ```
4.  **Push** to your branch:
    ```bash
    git push origin feature/my-new-feature
    ```
5.  Open a **Pull Request** against the `main` branch.

## ğŸ§  The V-Model Logic

This project automates the V-Model for systems engineering. When contributing, please keep the following in mind:

*   **Core Logic:** The core logic resides in `core/`. Avoid putting business logic directly in `app.py` (UI layer).
*   **Verification Engine:** The `VerificationEngine` class in `core/verification_engine.py` handles AI interactions.
*   **Generated Code:** Do not manually edit code generated by the AI in the `data/` directory unless necessary for debugging. The goal is for the AI to generate correct code.
